# 모바일 메뉴 디자인 옵션

## 현재 문제점
1. 모바일에서 화면이 100% 차지하지 않고 움직임
2. 플로팅 버튼이 일반적이지 않아 어색함

## 해결안 옵션

### 옵션 1: 상단 헤더에 햄버거 버튼 (가장 일반적) ⭐ 추천
**장점:**
- 웹사이트에서 가장 흔한 패턴
- 사용자가 익숙함
- 헤더와 통합되어 자연스러움

**구현:**
- 헤더 오른쪽에 햄버거 버튼 배치
- 클릭 시 왼쪽 또는 오른쪽에서 사이드 메뉴 슬라이드
- 설정 버튼은 메뉴 하단에 포함

**예시:**
```
[로고]                    [☰]
```

---

### 옵션 2: 하단 네비게이션 바 (앱 스타일)
**장점:**
- 모바일 앱처럼 직관적
- 주요 메뉴가 항상 보임
- 손가락으로 쉽게 접근

**구현:**
- 화면 하단에 고정된 네비게이션 바
- 아이콘 + 텍스트로 메뉴 표시
- 더보기 버튼으로 추가 메뉴 표시

**예시:**
```
┌─────────────────────────┐
│  📸    📝    🎨    ⋮   │
│ 사진  일기  작품  더보기│
└─────────────────────────┘
```

---

### 옵션 3: 상단 드롭다운 메뉴
**장점:**
- 간단하고 깔끔
- 공간 효율적

**구현:**
- 헤더에 햄버거 버튼
- 클릭 시 아래로 펼쳐지는 메뉴
- 설정은 메뉴 하단에

**예시:**
```
[로고]                    [☰]
         ┌─────────────┐
         │ 📸 사진첩   │
         │ 📝 일기장   │
         │ 🎨 작품활동 │
         │ ⚙️ 설정     │
         └─────────────┘
```

---

### 옵션 4: 왼쪽 사이드 드로어 (Material Design)
**장점:**
- Material Design 표준
- 많은 공간 활용 가능
- 설정과 메뉴를 함께 표시

**구현:**
- 헤더 왼쪽에 햄버거 버튼
- 왼쪽에서 슬라이드되는 전체 메뉴
- 프로필 정보도 함께 표시

**예시:**
```
[☰] [로고]              [설정]
┌─────────────────────┐
│ 👤 사용자 정보      │
│ ─────────────────── │
│ 📸 사진첩           │
│ 📝 일기장           │
│ 🎨 작품활동         │
│ ─────────────────── │
│ ⚙️ 설정             │
│ 🚪 로그아웃         │
└─────────────────────┘
```

## 화면이 100% 차지하지 않는 문제 해결

### 문제 원인:
1. `overflow-x` 설정 누락
2. `body` 또는 `html`의 `width` 제한
3. 고정 요소(fixed)가 레이아웃에 영향

### 해결 방법:

**1. globals.css에 추가:**
```css
html, body {
  width: 100%;
  max-width: 100%;
  overflow-x: hidden;
}

* {
  box-sizing: border-box;
}
```

**2. layout.tsx 수정:**
```tsx
<body className="... w-full overflow-x-hidden">
```

**3. 모바일 메뉴가 열릴 때 body 스크롤 방지:**
```tsx
useEffect(() => {
  if (isMobileMenuOpen) {
    document.body.style.overflow = 'hidden';
  } else {
    document.body.style.overflow = '';
  }
}, [isMobileMenuOpen]);
```
